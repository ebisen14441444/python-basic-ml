# 00. 環境準備

## この講習の標準環境

この講習では、基本的に **Google Colab** を使います。

Google Colab はブラウザ上で Python を実行できるサービスです。自分のPCに Python や VS Code を入れていなくても、同じ画面でコードを書いて実行できます。

まずは「環境構築を完璧にする」よりも、「Python のコードを読んで、少し変えて、実行できる」状態を目指します。

## 使うもの

- Google アカウント
- ブラウザ
- Google Colab
- 講習で配布されたノートブック

VS Code やローカルの Python 環境は必須ではありません。

## Colabを開く

講師から共有された Colab のリンクを開きます。

開いたら、画面上部のメニューから次を選び、自分用のコピーを作ります。

```text
ファイル > ドライブにコピーを保存
```

コピーを作ると、自分の Google Drive 上でノートブックを編集できるようになります。

## セルを実行する

Colab では、コードを書く場所を「セル」と呼びます。

コードセルに次のように書いて、左側の実行ボタンを押します。

```python
print("Hello, Python")
```

実行結果として `Hello, Python` と表示されればOKです。

セルは、上から順番に実行していくのが基本です。途中のセルを飛ばすと、後のセルで使う変数やデータがまだ作られていないことがあります。

## ライブラリを使う

この講習では、NumPy や Polars などのライブラリを使います。

NumPy は Colab ですぐ使えることが多いです。Polars が入っていない場合は、次のようにインストールします。

```python
!pip install polars
```

例えば、次のコードはそのまま実行できます。

```python
import polars as pl

data = {
    "name": ["Sato", "Suzuki", "Tanaka"],
    "score": [80, 65, 90],
}

df = pl.DataFrame(data)
print(df)
```

## ファイルを使う

CSV ファイルなどを使う場合は、Colab の左側にあるファイルアイコンからアップロードできます。

アップロードしたファイルは、ノートブックから次のように読み込めます。

```python
import polars as pl

df = pl.read_csv("scores.csv")
print(df.head())
```

アップロードしたファイルは、Colab の接続が切れると消えることがあります。必要なファイルは、配布されたノートブックや Google Drive に残しておくと安心です。

## よくあるつまずき

### セルを実行していない

変数を作るセルを実行する前に、その変数を使うセルを実行するとエラーになります。

エラーが出たら、まず上のセルから順番に実行し直してください。

### ファイルが見つからない

`FileNotFoundError` が出た場合は、CSV ファイルをアップロードできているか、ファイル名が合っているかを確認します。

例えば `scores.csv` を読み込みたい場合、ファイル名も同じ `scores.csv` になっている必要があります。

### 実行が終わらない

セルの左側にある実行中の表示が長く続く場合は、処理に時間がかかっているか、何かが止まっている可能性があります。

必要に応じて、画面上部のメニューから次を選んで実行を止めます。

```text
ランタイム > 実行を中断
```

この講習では、まず Colab で Python の基本に慣れることを優先します。ローカル環境の準備は、必要になったタイミングで扱います。
